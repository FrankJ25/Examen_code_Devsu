CONCLUSIONES Y HALLAZGOS

Durante la ejecución del proyecto se automatizaron flujos de pruebas funcionales para aplicaciones web y servicios REST, permitiendo validar el comportamiento esperado de los sistemas evaluados.

En las pruebas Web se verificaron procesos completos de compra en diferentes aplicaciones, incluyendo autenticación, selección de productos, gestión del carrito y finalización de pedidos. Los resultados demostraron que los flujos principales funcionan correctamente cuando se ingresan datos válidos.

En las pruebas de APIs se validaron diferentes operaciones sobre los servicios REST, como creación, consulta, actualización y eliminación de recursos. Se verificó que las APIs responden correctamente ante solicitudes válidas y retornan la información esperada en cada operación.

El uso de Serenity BDD con el patrón Screenplay permitió mantener una estructura clara y reutilizable del código de automatización. Por otro lado, Karate Framework facilitó la validación de servicios REST y la captura de las entradas y salidas de cada petición.

La automatización implementada permite ejecutar pruebas de forma repetitiva, reduciendo tiempos de validación manual y contribuyendo a asegurar la calidad del software mediante pruebas de regresión.
